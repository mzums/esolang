GardenersScript

A programming language inspired by gardening - manipulate "data plots" like a real gardener!

Introduction to GardenersScript

GardenersScript is an esoteric programming language (esolang) inspired by gardening. The programmer (gardener) manipulates "plots" that store values. The language includes instructions such as planting (plant), watering (water), pruning (prune), etc.

Language Philosophy

GardenersScript was designed with the following in mind:

  • A metaphorical approach to programming
  • Easy visualization of program state
  • Low entry barrier for beginners
  • Providing challenges for advanced programmers

Data Types

The language supports two basic data types:

  • Integers - basic numeric type
  • Strings - text surrounded by double quotes

Language Instructions

Below you will find a complete list of instructions available in GardenersScript:

plant

Creates a new plot or updates an existing one

plant <name> [value]

Example: plant x 10 or plant msg "Hello"

tend

Sets the current plot

tend <name>

Example: tend x

water

Adds a value to the current plot

water <value>

Example: water 5

prune

Subtracts a value from the current plot

prune <value>

Example: prune 3

graft

Multiplies the current plot by a value

graft <value>

Example: graft 2

divide

Divides the current plot by a value (integer division)

divide <value>

Example: divide 2

harvest

Adds the plot value to the program output

harvest [plot]

Example: harvest x or harvest (current plot)

check

Checks a condition for a plot

check <condition> <plot> <target>

Conditions: 'rich' (greater than), 'poor' (less than), 'balanced' (equal)

bloom

Starts a loop

bloom <count> <label>

Example: bloom 5 loop_start

wither

Decreases loop counter and returns to label

wither

function

Defines a new function

function <name> [arguments] { ... }

Example: function square [x] { ... }

call

Calls a function

call <name> [arguments]

Example: call square 5

Example Programs

Hello World

# Simplest program
plant msg "Hello, World!"
tend msg
harvest msg

Output: Hello, World!

Loop

# Displays numbers 0-4
plant i 0
plant max 5

loop_start:
    tend i
    harvest i
    water 1
    
    # Check if i < max (5)
    check poor i max
    bloom 1 loop_start

Output: 0 1 2 3 4

Function

# Squares a number
function square [x] {
    plant result 0
    tend x
    store        # Store x in greenhouse
    
    tend result
    retrieve     # Retrieve x into result (result = x)
    graft x     # Multiply by x (result = x * x)
    
    return result
}

plant num 5
call square num
harvest return

Output: 25

Basic operation

# Basic arithmetic operations
plant a 10
plant b 5

tend a
water 15     # a = 10 + 15 = 25
prune 7      # a = 25 - 7 = 18
harvest a

tend b
graft 3      # b = 5 * 3 = 15
divide 2     # b = 15 / 2 = 7 (integer division)
harvest b

Output: Hello, World!

Bloom/wither

# Countdown from 3 to 1 using bloom/wither
plant count 3
plant max 0

# Initialize loop counter
bloom 3 countdown  # Set loop count to 3

countdown:
    tend count
    harvest count  # Output: 3, 2, 1
    prune 1       # Decrement count
    
    wither        # Decrease loop counter and check

Output: 7

Graft

plant base 7
tend base
graft 3        # 7 * 3 = 21
harvest base   # Output: 21

Output: 21

Comparing

# Check stock levels
plant stock 25
plant min_stock 20
plant max_stock 100

# Check if stock is too low
check poor stock min_stock  # stock < min_stock?
harvest "WARNING: Low stock!"  # Skipped (25 > 20)

# Check if stock is too high
check rich stock max_stock  # stock > max_stock?
harvest "WARNING: Overstock!"  # Skipped (25 < 100)

# Check if stock is balanced
plant target 25
check balanced stock target
harvest "Stock at perfect level"  # This will execute

Output: Stock at perfect level

XOR

plant a 12
plant b 10
tend a
crossbreed b
harvest a

Output: 6

Language Syntax

General Rules

  • The program consists of instructions, one per line
  • Empty lines are ignored
  • Comments start with # and continue to the end of the line
  • Case insensitive for instructions

Variables (Plots)

  • Variable names can contain letters, digits and underscores
  • Cannot start with a digit
  • Declared using plant
  • Available throughout the program

Strings

  • Surrounded by double quotes: "text"
  • Support escape sequences: \n, \t, \\, \"

Functions

function <name> [param1, param2, ...] {
    # Function body
    return [value]
}

Control Structures

# Loop
bloom <iterations> <label>

# Label
label_name:

# Conditional
check <condition> <plot> <target>
# If condition not met, skips next instructions

Advanced Features

Greenhouse (Stack)

Ability to store and retrieve values from a stack:

  • store - saves the value of the current plot
  • retrieve - retrieves a value from the stack to the current plot

Bitwise Operations

Advanced bit operations:

  • crossbreed <value> - XOR with the current plot

Error Handling

The interpreter reports errors in cases of:

  • Use of undefined variable
  • Use of unknown instruction
  • Division by zero
  • No current plot for operations
  • Type errors (e.g., numeric operations on strings)
